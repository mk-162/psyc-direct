# Cocoon Site Audit — March 2026

## Executive Summary

The Cocoon site is in good shape overall. **Design system ✓**, **TinaCMS integrated ✓**, **Build/deploy working ✓**. Main polish needed: cleaning up inline styles and a few UX improvements.

---

## Findings

### 🔴 High Priority

#### 1. Inline Styles in Block Components (Known Issue)
**Files:** `Hero.tsx`, `FullWidthImage.tsx`

Two components still use inline styles instead of CSS classes:

```tsx
// Hero.tsx — dynamic background image
style={bgStyle}

// FullWidthImage.tsx — dynamic height
style={data.height ? { height: data.height, objectFit: 'cover', width: '100%' } : undefined}
```

**Impact:** Breaks design system consistency, harder to maintain  
**Fix:** Add utility classes to globals.css and reference those

**Recommended classes to add:**
```css
/* In globals.css */
.hero-bg-dynamic { background-size: cover; background-position: center; }
.img-height-auto { height: auto; }
.img-height-400 { height: 400px; }
.img-height-50vh { height: 50vh; }
.img-height-75vh { height: 75vh; }
```

---

### 🟡 Medium Priority

#### 2. TinaCMS Admin Routing Bug
**Issue:** Clicking tabs in `/admin` creates duplicate paths:
```
/admin/index.html#/~/admin/index.html#/~
```

**Root cause:** Hash-based routing is appending relative paths  
**Impact:** Cosmetic — doesn't break editing but looks buggy  
**Fix:** Check Tina config `router` functions or add base path config

#### 3. SEO Meta Tags Not Dynamic
**File:** `layout.tsx`

Meta title/description are hardcoded:
```tsx
title: "Cocoon Healthcare - Premium Wellness & Preventive Health",
description: "Proactive healthcare..."
```

**Fix:** Child pages should override via Next.js `metadata` export. Most pages likely don't.

#### 4. Missing Type Safety on CMS Data
**Files:** Block components use `any` types:
```tsx
component = <Hero data={block as any} />;
```

**Impact:** No IntelliSense, easier to break  
**Fix:** Define block data interfaces in `types.ts`

---

### 🟢 Low Priority (Polish)

#### 5. Unused Block Import
**File:** `PremiumPageRenderer.tsx`
```tsx
// import { IntroGallery } from './IntroGallery'; // Removed per feedback
```

Clean up — it's commented out but the comment is stale.

#### 6. `any[]` in Tina Config
**File:** `tina/config.ts` uses `as any` in several places:
```tsx
fields: [...] as any,
```

Should use proper TypeScript types.

#### 7. Console Warn in PremiumPageRenderer
```tsx
console.warn(`Block not mapped: ${block.__typename}`);
```

This fires in production. Should be dev-only.
